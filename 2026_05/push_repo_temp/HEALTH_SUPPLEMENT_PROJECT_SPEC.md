# 身体健康监测与补给推荐网页项目说明

## 1. 项目定位

本项目是一个面向普通用户的健康状态问卷与营养补给建议网页。用户通过填写基础信息、健康状况、生活习惯和饮食偏好，获得个性化的营养关注建议、生活方式建议和安全提醒。

本项目不是医疗诊断工具，不提供疾病诊断、治疗方案或药物建议。所有推荐结果应以“可能需要关注的营养素”或“建议咨询专业人士”为表达方式，避免使用“必须服用”“可以治疗”“保证改善”等确定性医疗表述。

## 2. 核心用户流程

1. 用户进入首页，查看项目说明与安全提示。
2. 用户点击开始评估。
3. 用户填写基本身份信息。
4. 用户填写既往健康状况和用药风险信息。
5. 用户填写生活习惯、饮食习惯和身体感受问卷。
6. 系统根据规则计算不同健康维度的关注等级。
7. 系统生成补给建议、生活方式建议和安全提醒。
8. 用户可重新填写问卷或保存/导出结果。

## 3. 页面模块规划

### 3.1 首页

功能：

- 展示产品名称和一句话介绍。
- 说明本工具仅用于健康与营养参考。
- 提供“开始评估”按钮。
- 展示隐私说明：当前版本不上传用户数据。

建议文案：

> 通过简短问卷了解你的生活习惯、饮食结构和健康风险，生成个性化营养关注建议。

### 3.2 基本信息页

字段：

- 性别：男 / 女 / 其他 / 不愿透露
- 年龄段：18 岁以下 / 18-30 / 31-45 / 46-60 / 60 岁以上
- 身高：可选
- 体重：可选
- 饮食类型：普通饮食 / 素食 / 纯素 / 控糖饮食 / 低盐饮食 / 高蛋白饮食 / 其他
- 特殊状态：怀孕 / 备孕 / 哺乳 / 无 / 不愿透露

用途：

- 用于判断基础营养关注点。
- 用于触发安全提示，例如孕期、未成年人、老年用户需谨慎使用补剂。

### 3.3 健康状况筛查页

问题建议：

- 是否有糖尿病或血糖异常？
- 是否有高血压、高血脂或心脏疾病？
- 是否有肝病、肾病或甲状腺疾病？
- 是否曾被医生告知贫血、缺铁或维生素 D 缺乏？
- 是否正在长期服药？
- 是否正在服用抗凝药、降糖药、降压药或降脂药？
- 是否对鱼类、坚果、乳制品、大豆或其他食物过敏？

选项：

- 是
- 否
- 不确定
- 不愿透露

用途：

- 作为安全拦截条件。
- 当用户存在疾病、用药、怀孕、哺乳等情况时，补剂建议应降级为“请先咨询医生或营养师”。

### 3.4 生活习惯问卷页

频率选项：

- 从不
- 偶尔
- 有时
- 经常
- 几乎总是

问题建议：

- 我经常熬夜或睡眠不足。
- 我白天容易疲劳、注意力不集中。
- 我长时间使用电脑或手机，并感到眼睛干涩、酸胀或疲劳。
- 我很少晒太阳。
- 我很少吃新鲜蔬菜和水果。
- 我很少吃鱼类、坚果或富含优质脂肪的食物。
- 我经常吃甜食、含糖饮料或精制碳水。
- 我经常吃重盐、腌制、油炸或加工食品。
- 我经常抽烟或接触二手烟。
- 我经常饮酒。
- 我运动量不足。
- 我近期压力较大，恢复感较差。
- 我肠胃不规律，或膳食纤维摄入较少。

### 3.5 结果页

结果页应包含：

- 健康画像摘要
- 重点关注维度
- 推荐关注的营养素
- 饮食与生活方式建议
- 安全提醒
- 重新评估按钮

推荐结果表达示例：

- “你可能需要关注维生素 D 摄入。”
- “建议优先通过饮食、作息和体检确认营养状态。”
- “如果你正在服药或有慢性疾病，请先咨询医生后再使用补剂。”

不建议表达：

- “你缺乏维生素 D。”
- “你必须服用某补剂。”
- “该补剂可以治疗糖尿病/心脏病/失眠。”

## 5. 推荐规则设计

### 5.1 健康维度

系统可将问卷答案汇总到以下维度：

- 睡眠与疲劳
- 眼部疲劳
- 骨骼与维生素 D 风险
- 蔬果与抗氧化摄入
- 心血管饮食风险
- 控糖饮食风险
- 肠道与膳食纤维
- 素食营养缺口
- 压力与恢复
- 补剂安全风险

### 5.2 推荐映射

| 触发条件 | 关注营养素/补给方向 | 说明 |
| --- | --- | --- |
| 长时间用眼、眼疲劳分高 | 叶黄素、玉米黄质、Omega-3、维生素 A 食物来源 | 不替代眼科检查 |
| 经常熬夜、疲劳分高 | B 族维生素、镁、优质蛋白 | 优先改善睡眠 |
| 很少晒太阳 | 维生素 D、钙 | 建议结合检测结果 |
| 很少吃鱼类和坚果 | Omega-3、坚果类优质脂肪 | 鱼类过敏用户需避开鱼油 |
| 蔬果摄入少 | 维生素 C、膳食纤维、多酚类食物 | 优先从饮食补充 |
| 甜食或含糖饮料多 | 膳食纤维、低 GI 饮食建议 | 糖尿病用户不做治疗建议 |
| 重盐和加工食品多 | 低钠饮食、钾来源食物 | 肾病用户需医生确认 |
| 素食或纯素 | 维生素 B12、铁、锌、Omega-3 | 建议定期检查相关指标 |
| 抽烟或二手烟暴露 | 维生素 C 食物来源、抗氧化饮食 | 避免高剂量补剂承诺 |
| 压力大、恢复差 | 镁、B 族、规律运动和睡眠 | 不宣称治疗焦虑或抑郁 |

### 5.3 关注等级

每个维度可计算分数：

- 0-2：低关注
- 3-5：中关注
- 6 及以上：高关注

具体分数可根据问题数量调整。后续分支如新增问题，应同步更新维度计算规则。

## 6. 安全规则

以下用户应显示强安全提醒：

- 未成年人
- 60 岁以上用户
- 怀孕、备孕或哺乳用户
- 有糖尿病、心脏疾病、高血压、高血脂、肝病、肾病用户
- 正在长期服药用户
- 正在使用抗凝药、降糖药、降压药、降脂药用户
- 有严重食物过敏用户

安全提醒示例：

> 你的问卷中包含可能影响补剂选择的健康因素。请在医生或注册营养师指导下选择补剂，避免自行叠加高剂量产品。

通用安全提醒：

- 补剂不能替代均衡饮食、睡眠、运动和正规治疗。
- 脂溶性维生素 A、D、E、K 不宜长期超量补充。
- 同时服用多种复合维生素可能导致重复摄入。
- 有慢性疾病或正在服药时，应先咨询医生。

## 7. 前端实现建议

推荐先使用静态单页结构：

```text
index.html
style.css
script.js
HEALTH_SUPPLEMENT_PROJECT_SPEC.md
```

实现方式：

- 使用 HTML 搭建页面结构。
- 使用 CSS 实现健康、可信、清爽的视觉风格。
- 使用 JavaScript 保存问卷状态、计算分数、生成结果。
- 暂不接入后端，用户数据仅保存在浏览器内存中。

后续可升级：

- React 或 Vue 单页应用。
- 本地存储历史结果。
- 用户登录和云端记录。
- 管理后台维护问题库与推荐规则。
- 推荐规则配置化。
- 生成 PDF 或图片版结果报告。

## 8. 分支开发建议

为避免未来功能合并冲突，建议按模块拆分分支：

| 分支方向 | 建议命名 | 负责内容 |
| --- | --- | --- |
| 页面结构 | feature/page-flow | 首页、问卷页、结果页结构 |
| 样式设计 | feature/ui-style | CSS、响应式布局、组件样式 |
| 问卷数据 | feature/questionnaire | 问题库、选项、字段定义 |
| 推荐算法 | feature/recommendation-rules | 分值计算、维度映射、推荐生成 |
| 安全提示 | feature/safety-guardrails | 慢病、用药、孕期等安全规则 |
| 报告导出 | feature/report-export | 导出 PDF、图片或文本结果 |
| 数据持久化 | feature/storage | localStorage 或后端接口 |

合并约定：

- 页面结构分支不要直接修改推荐规则。
- 推荐规则分支不要直接修改大范围样式。
- 问卷字段新增后，应同步更新数据结构和推荐规则。
- 所有用户可见的健康建议都必须保留安全提醒。
- 合并前检查移动端布局和基础问卷流程。

## 9. 可验收标准

基础版本完成时应满足：

- 用户可以完整完成问卷。
- 问卷进度清晰可见。
- 不同答案会生成不同推荐结果。
- 高风险健康状态会触发安全提醒。
- 页面在桌面端和移动端都能正常使用。
- 不出现诊断性或治疗性承诺文案。

## 10. 后续可扩展功能

- 多语言切换：中文 / 英文。
- 根据年龄和性别调整问题优先级。
- 添加 BMI 和运动建议。
- 添加结果图表。
- 添加体检指标输入，例如血糖、血脂、维生素 D、铁蛋白。
- 添加补剂冲突提醒。
- 添加饮食优先建议，将补剂作为次级方案。
- 添加“给医生或营养师查看”的报告模式。
